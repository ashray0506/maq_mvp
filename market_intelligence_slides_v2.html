Data quality & governance
14 rules across three layers. Every failure logged. Every record accounted for. Built to post-trade audit standards from the start.
BronzeIngest
B1Retry 3× with 5s backoff
B2AV rate limit via JSON key — not HTTP status
B3Schema validation before writing
B4FRED "." → null with count logged
SilverTransform
S1Null check → quarantine rows
S2Completeness ≥ 60 rows
S3No future dates → quarantine
S4Join ≥ 3 shared months or halt
S5All dates → UTC
GoldMetrics
G1RSI null first 14 rows — window warmup
G2VWAP excludes zero-volume rows
G3≥ 3 months macro before EMA/SMA
G4Sharpe/Vol null first 20 rows
G5Sharpe uses FEDFUNDS/252 as risk-free rate
audit_pipeline_runs
rows in/out/quarantined per step per run
audit_dq_results
PASS/FAIL per rule per run with detail
quarantine_records
failed records with rule ID + raw record — never silently dropped
"A record either passes and moves forward, or is explicitly accounted for in the quarantine table. There is no third option where data disappears silently." — the pattern you'd want in any post-trade reconciliation layer.
